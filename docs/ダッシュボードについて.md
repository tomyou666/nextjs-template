# ダッシュボードについて

## 概要

ダッシュボードは、アプリケーションの管理画面として機能し、各種機能へのアクセスを提供します。

## ディレクトリ構造

ダッシュボード関連のファイルは以下のディレクトリに配置されます：

```text
src/
├── app/
│   └── dashboard/        # ダッシュボード関連ページ
├── components/           # 再利用可能なコンポーネント
└── lib/                  # ユーティリティと共有ロジック
```

## ダッシュボードの追加手順

### 1. ルートの定義

`src/lib/share/constants.ts`に新しいダッシュボードのルートを追加します：

```typescript
export const ROUTES: Routes = {
  dashboard: {
    path: '/dashboard',
    label: 'ダッシュボード',
    children: {
      // 既存のルート
      index: {
        path: '/dashboard',
        label: 'Dashboard',
      },
      // 新しいルートを追加
      newFeature: {
        path: '/dashboard/new-feature',
        label: '新機能',
      },
    },
  },
}
```

### 2. ページの作成

`src/app/dashboard/`配下に新しいページを作成します：

```typescript
// src/app/dashboard/new-feature/page.tsx
import { MarkdownHelpDialog } from '@/components/MarkDownHelpDIalog'

export default function NewFeaturePage() {
  return (
    <div className="flex-1 space-y-4 p-8 pt-6">
      <div className="flex items-center justify-between space-y-2">
        <h2 className="font-bold text-3xl tracking-tight">新機能</h2>
        <MarkdownHelpDialog
          title="新機能について"
          content={newFeatureHelpContent}
          ariaLabel="新機能についてのヘルプ"
        />
      </div>
      {/* ページのコンテンツ */}
    </div>
  )
}
```

### 3. コンポーネントの作成

必要に応じて、`src/components/`配下に新しいコンポーネントを作成します。

### 4. 認証の設定

ダッシュボードは認証が必要なため、`src/middleware.ts`で保護されていることを確認します：

```typescript
export const config = {
  matcher: [
    // 全体的な保護
    '/((?!api|_next/static|_next/image|favicon.ico|login|signup|$).*)',
    // ダッシュボード配下の保護
    '/dashboard/:path*',
  ],
}
```

## 注意事項

- 必要に応じてヘルプコンテンツを追加してください
